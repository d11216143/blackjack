# 10點半（Ten-and-a-half）單檔網頁遊戲開發規格（Spec）

> 目標：依照前述規則，製作**單檔**（僅 `index.html`）即可執行的「10點半」遊戲。  
> 形式：純前端、離線可玩、無需伺服器、無外部依賴（建議不使用 CDN）。  
> 範圍：1 位玩家 vs 系統莊家（Banker/Dealer），含基本 UI 與簡要說明。

---

## 1. 遊戲規則（Rule Requirements）

### 1.1 牌組
- 使用一副標準撲克牌 52 張（不含 Joker）。
- 每局開始必須重新洗牌（或牌堆不足時自動重洗）。

### 1.2 點數計算（10點半）
- A～10：點數等於牌面數字（A=1, 2=2 … 10=10）
- J/Q/K：每張 **0.5 點**
- 手牌點數為所有牌點數加總。
- **爆牌（Bust）**：手牌點數 **> 10.5** 即爆牌，立即判輸（玩家回合直接結束）。

### 1.3 回合流程（單人對莊）
1. 進入新局（Round Start）
2. 玩家回合（Player Turn）
   - 初始發牌：玩家 1 張、莊家 1 張（莊家牌可顯示或半遮罩，見 UI 規格）
   - 玩家可重複選擇：
     - **要牌（Hit）**：再抽 1 張
     - **停牌（Stand）**：結束玩家回合
   - 玩家若爆牌：直接進入結算（玩家輸）
3. 莊家回合（Banker Turn）
   - 在玩家未爆牌的前提下才執行
   - 莊家採用固定策略（硬規則）：
     - **莊家點數 < 8 必須要牌**
     - **莊家點數 ≥ 8 必須停牌**
   - 莊家爆牌則莊家輸（玩家贏）
4. 結算（Settlement）
   - 若玩家爆牌：玩家輸
   - 若莊家爆牌：玩家贏
   - 若皆未爆牌：
     - 玩家點數 > 莊家點數：玩家贏
     - 玩家點數 < 莊家點數：玩家輸
     - 玩家點數 = 莊家點數：**和局（Push）**
5. 顯示本局結果與原因，提供「下一局」按鈕。

> 註：以上策略與平手規則為此單檔版本的預設值；若需改為「莊贏」或不同莊家門檻，須在本規格另列變更。

## 2. 功能需求（Functional Requirements）

### 2.1 基本功能
- [F-01] 新局：重置狀態、洗牌、發起始牌。✅
- [F-02] 要牌：玩家按下 Hit 後抽 1 張並更新點數/狀態。✅
- [F-03] 停牌：玩家按下 Stand 後轉入莊家回合。✅
- [F-04] 自動莊家回合：依 1.3 規則自動抽牌直到停牌或爆牌。✅
- [F-05] 結算：依 1.3 規則判斷勝負，顯示結果文字。✅
- [F-06] 操作鎖定：在非玩家回合時，禁用 Hit/Stand；結算後禁用 Hit/Stand 直到下一局。✅
- [F-07] 牌面呈現：顯示玩家與莊家的手牌列表（文字或簡易卡片 UI）。✅
- [F-08] 點數呈現：即時更新玩家與莊家點數（莊家可在結算前隱藏總點數，見 UI）。✅

### 2.2 規則說明（簡單敘述）
- [F-09] UI 需包含「規則簡述區塊」，可折疊或固定顯示，至少包含：✅
  - J/Q/K=0.5 ✅
  - 目標 10.5、超過爆牌 ✅
  - 玩家可要牌或停牌 ✅
  - 莊家策略（<8 要牌，≥8 停牌）✅
  - 勝負判定（含平手和局）✅

### 2.3 不包含（Out of Scope）
- 多人連線/房間/聊天
- 下注、籌碼、金流
- 帳號登入
- 動畫特效（可選）

## 3. 使用者介面規格（UI Requirements）

### 3.1 單檔結構
- 必須只有一個檔案：`index.html` ✅
- `index.html` 內同時包含：
  - HTML 結構 ✅
  - CSS（`<style>`）✅
  - JavaScript（`<script>`）✅
- 不依賴外部檔案（圖片可用 Unicode/文字取代）。✅

### 3.2 必要 UI 元件
- 標題區 ✅
  - 顯示：`10點半（Ten-and-a-half）` ✅
- 規則簡述區（可用 `<details>`）✅
- 遊戲區 ✅
  - 玩家區塊 ✅
    - 手牌顯示（例如以卡片樣式 div 列出）✅
    - 玩家點數顯示（例如：`Player: 8.5`）✅
  - 莊家區塊 ✅
    - 手牌顯示 ✅
    - 莊家點數顯示：✅
      - 玩家回合時可顯示「部分資訊」：例如只顯示第一張牌與「?」或不顯示總點數 ✅
      - 結算後顯示完整點數 ✅
- 操作按鈕 ✅
  - [UI-01] `New Game / 下一局` ✅
  - [UI-02] `Hit / 要牌` ✅
  - [UI-03] `Stand / 停牌` ✅
- 訊息區（狀態與結果）✅
  - 顯示目前階段：玩家回合/莊家回合/結算 ✅
  - 顯示本局結果：玩家贏/輸/和局與原因（爆牌、點數比較）✅

### 3.3 最低可用外觀（基本 UI）
- 需有基本排版（區塊分區、字體、間距）✅
- 手牌以「卡片」呈現（簡易即可）✅
- 不要求 RWD，但版面不可嚴重跑版 ✅

## 4. 程式設計規格（Technical / SD Lite）

### 4.1 建議資料結構
- `Card` ✅
  - `suit`: `"♠" | "♥" | "♦" | "♣"` ✅
  - `rank`: `"A" | "2" ... "10" | "J" | "Q" | "K"` ✅
- `State` ✅
  - `deck: Card[]` ✅
  - `playerHand: Card[]` ✅
  - `bankerHand: Card[]` ✅
  - `phase: "PLAYER_TURN" | "BANKER_TURN" | "SETTLED"` ✅
  - `result: { outcome: "WIN"|"LOSE"|"PUSH"|null, reason: string }` ✅

### 4.2 主要函式（必備）
- `buildDeck(): Card[]` ✅
- `shuffle(deck: Card[]): Card[]`（Fisher–Yates）✅
- `draw(state): Card`（從 deck pop/shift）✅
- `cardValue(card): number` ✅
- `handPoints(hand): number` ✅
- `isBust(points): boolean` ✅
- `startRound()` ✅
- `playerHit()` ✅
- `playerStand()` ✅
- `bankerAutoPlay()`（依 <8 hit）✅
- `settle()` ✅
- `render()`（單一渲染/更新 UI 的入口）✅

### 4.3 約束
- 所有抽牌/結算邏輯必須在 JS 內完成，**不得由 UI 文字推導結果**。✅
- Hit/Stand 在不合法時（非玩家回合）必須無效且按鈕 disabled。✅

## 7. 實現細節（Implementation Details）

### 7.1 技術棧
- 純 HTML5 + CSS3 + Vanilla JavaScript
- 無外部依賴、無 CDN
- 支援離線運行

### 7.2 核心功能實現
1. **牌組管理**
   - 使用 Fisher-Yates 演算法確保公平洗牌
   - 牌堆不足時自動重新洗牌

2. **遊戲狀態管理**
   - 使用狀態機模式管理遊戲階段
   - 階段：INIT → PLAYER_TURN → BANKER_TURN → SETTLED

3. **UI 渲染**
   - 卡片使用 CSS 樣式美化
   - 紅色花色（♥♦）與黑色花色（♠♣）區分
   - 莊家第二張牌在玩家回合時隱藏

4. **莊家邏輯**
   - 完全自動化，按規則執行
   - 使用延遲動畫提升用戶體驗

### 7.3 測試場景
已驗證以下場景：
- ✅ 玩家爆牌（點數 > 10.5）
- ✅ 莊家爆牌（玩家獲勝）
- ✅ 玩家點數高於莊家
- ✅ 莊家點數高於玩家
- ✅ 點數相同（和局）
- ✅ J/Q/K 正確計為 0.5 點
- ✅ 莊家自動策略（< 8 要牌，≥ 8 停牌）
